# Add Copy Buttons to Inputs, Prompts, and Results

**Date:** 2026-02-20 14:09
**Task:** Add copy-to-clipboard buttons on input textareas, prompt textareas, and result cells

## Goal
Each editable textarea (inputs, prompts) and each result cell should have a small copy icon button that copies the content to clipboard. The button should show a checkmark briefly after copying.

## Approach

### Step-by-step plan
1. Create a reusable `CopyButton` component in `components/copy-button.tsx` using lucide-react Copy/Check icons
2. Add copy button to the input textarea area (positioned inside the textarea container, top-right)
3. Add copy button to the prompt textarea area (same position)
4. Add copy button to result cells in `ResultCellContent`

### Key decisions
- **Reusable component:** Single `CopyButton` component takes a `value` string prop
  - Alternatives: inline the copy logic in each spot — rejected, DRY violation
- **Positioning:** Absolute positioned in top-right of each container, small and gray
- **Feedback:** Swap icon to Check for ~1.5s after copy
- **No shadcn clipboard component exists** — build a minimal one with lucide icons + button styling

## Architectural Considerations
- `CopyButton` goes in `components/` — it only imports from `lib/` (cn utility), follows zone rules
- No new dependencies needed — lucide-react already available

## Risks & Edge Cases
- Clipboard API requires HTTPS or localhost — both apply here
- Empty content: hide the button when value is empty

## Files to Modify
- `apps/web/src/components/copy-button.tsx` — new reusable copy button component
- `apps/web/src/app/page.tsx` — add copy buttons to input and prompt textarea containers
- `apps/web/src/components/result-cell.tsx` — add copy button to result cells
